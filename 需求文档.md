## 1. 项目背景与目标

### 1.1 项目背景
五子棋（Gomoku / Five in a Row）是一种经典的两人对弈策略棋类游戏，规则简单、对局节奏快、上手门槛低，拥有广泛的用户基础。本项目旨在开发一款运行于 Web 浏览器中的五子棋游戏，提供流畅的对弈体验与简洁美观的界面，作为轻量级休闲娱乐产品。

### 1.2 项目目标
- **业务目标**：提供一款可直接在浏览器中运行、无需安装的五子棋游戏，满足用户碎片化娱乐需求。
- **产品目标**：
  - 支持人人对战与人机对战两种核心模式。
  - 界面简洁、交互流畅、规则判定准确。
  - 首屏加载时间 ≤ 2 秒（常规网络环境）。
- **技术目标**：纯前端实现，零后端依赖，可静态部署。

### 1.3 非目标（本期不做）
- 多人在线联网对战（需要后端信令服务）。
- 用户账号体系与对局数据云端存储。
- 排行榜、成就系统、社交分享等运营功能。
- 棋谱导出 / 复盘分析等高级功能。

## 2. 用户角色与典型场景

### 2.1 用户角色
| 角色 | 说明 |
| --- | --- |
| 普通玩家 | 通过浏览器打开游戏进行对弈的终端用户，无需注册登录 |

### 2.2 典型场景
1. **单人消遣**：玩家选择「人机对战」模式，与 AI 对弈消磨时间。
2. **双人对弈**：两人共用一台设备，选择「人人对战」模式轮流落子。
3. **快速重开**：对局结束后，玩家点击「再来一局」快速开始新对局。

## 3. 功能需求

### 3.1 功能清单总览
| 编号 | 功能模块 | 优先级 |
| --- | --- | --- |
| F-01 | 游戏模式选择 | P0 |
| F-02 | 棋盘渲染与交互 | P0 |
| F-03 | 落子与回合切换 | P0 |
| F-04 | 胜负判定 | P0 |
| F-05 | 游戏状态控制（认输 / 悔棋 / 重开） | P0 |
| F-06 | 人机对战 AI | P1 |
| F-07 | 落子提示与高亮 | P1 |
| F-08 | 计时与步数统计 | P2 |
| F-09 | 主题切换（明 / 暗） | P2 |

### 3.2 详细功能描述

#### F-01 游戏模式选择
- **描述**：玩家进入游戏后，在主界面选择对局模式。
- **输入**：玩家点击模式按钮。
- **输出**：进入对应模式的对局界面。
- **规则**：
  - 提供「人人对战」「人机对战」两种模式入口。
  - 人机对战模式下，需让玩家选择执黑（先手）或执白（后手），并选择 AI 难度（简单 / 普通 / 困难）。
  - 模式选择后即开始对局，无需额外确认步骤。

#### F-02 棋盘渲染与交互
- **描述**：在页面中渲染标准 15×15 五子棋棋盘，并响应玩家点击。
- **规则**：
  - 棋盘为 15 行 × 15 列共 225 个交叉点。
  - 黑子先行，双方轮流在空白交叉点落子。
  - 鼠标悬停时显示半透明预览子，指示当前落子位置。
  - 已有棋子的位置不可再次落子，点击无效。
  - 落子后棋子以平滑动画出现在对应交叉点。
  - 棋盘自适应窗口大小，保证在不同分辨率下均可见且可操作。

#### F-03 落子与回合切换
- **描述**：管理落子流程与当前回合状态。
- **规则**：
  - 维护当前回合方（黑 / 白），落子后自动切换。
  - 落子后立即进行胜负判定，若未分胜负则等待下一手。
  - 人机对战模式下，玩家落子后由 AI 计算并自动落子。
  - 整局棋谱以数组形式在内存中保存，支持后续悔棋与重开。

#### F-04 胜负判定
- **描述**：每次落子后判断是否形成五子连珠。
- **规则**：
  - 判定范围：以最新落子点为中心，检查横、竖、正斜（↘）、反斜（↙）四个方向。
  - 任一方向上连续同色棋子数 ≥ 5 即判定该方获胜。
  - 判定胜利后高亮显示连成的五子（或更多子）。
  - 棋盘填满仍无人获胜时判定为和棋。
  - 胜负判定完成后，弹出对局结果提示（胜 / 负 / 和），并提供「再来一局」「返回主菜单」操作。

#### F-05 游戏状态控制
- **描述**：提供对局过程中的辅助操作。
- **子功能**：
  - **认输**：当前回合方主动认输，对方获胜，弹出结果提示。
  - **悔棋**：
    - 人人对战：悔棋一步，回到上一回合。
    - 人机对战：悔棋两步（玩家与 AI 各一步），保证回合仍由玩家进行。
    - 对局开始前不可悔棋；无历史步数时按钮置灰。
  - **重新开始**：清空棋盘与历史，保留当前模式与设置，开始新一局。
  - **返回主菜单**：清空当前对局，回到模式选择界面。

#### F-06 人机对战 AI
- **描述**：人机对战模式下的 AI 对手逻辑。
- **规则**：
  - AI 难度分为「简单 / 普通 / 困难」三档：
    - **简单**：随机选择空位落子，仅避免明显失误。
    - **普通**：基于启发式评分，能识别活三、冲四等常见棋型并进行攻防。
    - **困难**：在普通基础上加深搜索层级（如极小极大 + Alpha-Beta 剪枝），具备更强的预判能力。
  - AI 落子应在合理时间内完成（普通 ≤ 500ms，困难 ≤ 1500ms）。
  - AI 落子前显示「思考中…」状态提示。

#### F-07 落子提示与高亮
- **描述**：增强对弈过程的可读性。
- **规则**：
  - 最新一手棋子以高亮标记（如描边或星标），便于识别。
  - 鼠标悬停时显示预览子与坐标提示（如 H8）。
  - 胜负判定后高亮连珠的五子。

#### F-08 计时与步数统计
- **描述**：显示对局时长与双方步数。
- **规则**：
  - 对局开始时启动计时器，实时显示已用时（mm:ss）。
  - 分别统计黑白双方落子步数并显示。
  - 对局结束（胜负 / 认输 / 重开）时停止计时。

#### F-09 主题切换
- **描述**：提供明暗两套视觉主题。
- **规则**：
  - 默认跟随系统主题，并提供手动切换按钮。
  - 切换后主题偏好记录至 localStorage，下次访问保持一致。

## 4. 界面与交互需求

### 4.1 页面结构
1. **主菜单页**：游戏标题、模式选择按钮、（人机模式下的）执子与难度选择。
2. **对局页**：棋盘区 + 信息侧栏（当前回合、步数、计时、操作按钮）。
3. **结果弹窗**：对局结束时展示胜负结果与后续操作入口。

### 4.2 交互原则
- 所有操作通过鼠标点击完成，避免键盘依赖。
- 关键操作（认输、返回主菜单）需二次确认，防误触。
- 落子响应即时，无明显延迟感。
- 界面元素在不同分辨率下保持合理布局，移动端可横屏使用。

### 4.3 视觉风格
- 整体风格简洁现代，棋盘采用经典木纹或素色背景。
- 黑白棋子对比清晰，边界分明。
- 配色遵循 WCAG AA 对比度标准，确保可读性。

## 5. 非功能需求

| 类别 | 要求 |
| --- | --- |
| 性能 | 首屏加载 ≤ 2s；单次落子响应 ≤ 100ms；AI 普通难度思考 ≤ 500ms |
| 兼容性 | 兼容 Chrome / Edge / Firefox / Safari 最新两个主版本；支持移动端浏览器 |
| 响应式 | 适配 1280×720 及以上桌面分辨率；适配主流移动设备横屏 |
| 可维护性 | 代码模块化分层（UI / 逻辑 / AI / 状态管理）；核心逻辑与渲染解耦 |
| 可部署性 | 构建产物为静态文件，可部署至任意静态服务器或对象存储 |
| 安全性 | 纯前端运行，无网络请求与外部依赖风险；无 XSS 风险输入 |
| 可访问性 | 关键操作支持键盘聚焦；颜色对比度满足 WCAG AA |

## 6. 技术选型建议

| 层面 | 建议 | 说明 |
| --- | --- | --- |
| 框架 | React 或 Vue 3 | 组件化开发，状态管理清晰 |
| 语言 | TypeScript | 类型安全，降低维护成本 |
| 构建 | Vite | 启动快、构建产物体积小 |
| 样式 | CSS Modules / Tailwind CSS | 原子化样式，便于主题切换 |
| 状态管理 | React: Zustand / Vue: Pinia | 軻量级，适合本项目规模 |
| AI 实现 | 纯 TypeScript 自研 | 评分函数 + 极小极大剪枝，无需外部依赖 |
| 部署 | 静态托管（如 Nginx / 对象存储） | 无后端依赖 |

## 7. 验收标准

| 编号 | 验收项 | 验收标准 |
| --- | --- | --- |
| AC-01 | 模式选择 | 主菜单可正常进入两种模式，人机模式可选执子与难度 |
| AC-02 | 棋盘渲染 | 15×15 棋盘正确渲染，悬停有预览，落子有动画 |
| AC-03 | 落子规则 | 黑先白后，已占位不可落子，落子后回合自动切换 |
| AC-04 | 胜负判定 | 横竖斜四方向五连珠均能正确判胜，和棋判定正确 |
| AC-05 | 状态控制 | 认输、悔棋、重开、返回主菜单功能均符合规则定义 |
| AC-06 | AI 对弈 | 三档难度均可正常对弈，思考时间在限定范围内 |
| AC-07 | 计时统计 | 计时与步数统计准确，对局结束停止计时 |
| AC-08 | 主题切换 | 明暗主题可切换并持久化，切换后界面正常 |
| AC-09 | 兼容性 | 在要求浏览器与分辨率下界面与功能正常 |
| AC-10 | 性能 | 首屏加载与落子响应时间满足非功能要求 |

## 8. 术语表

| 术语 | 说明 |
| --- | --- |
| 五子棋 / Gomoku | 两人对弈棋类，先将五枚同色棋子在横、竖或斜方向连成一线者获胜 |
| 五连珠 | 同色棋子在任一方向连续排列达到五个或以上 |
| 活三 | 同色三连且两端均未被阻挡，下一手可形成冲四 |
| 冲四 | 同色四连且一端被挡，下一手即可获胜的威胁棋型 |
| 极小极大算法 | 博弈树搜索算法，一方追求收益最大化，另一方追求收益最小化 |
| Alpha-Beta 剪枝 | 对极小极大搜索的优化，剪除不影响决策的分支以提升效率 |
